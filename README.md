# Projeto Lógico de Banco de Dados - Oficina Mecânica
Este repositório contém a solução para o desafio de projeto da DIO, focado na modelagem e criação de um banco de dados para o gerenciamento de ordens de serviço em uma oficina.
# Escopo do Projeto:
Clientes e Veículos: Cadastro de clientes e a relação com seus respectivos veículos.
Ordem de Serviço (OS): Controle de emissão de OS, com data, valor total e status do serviço.
Mão de Obra e Peças: Estrutura para separar o que é custo de serviço (mecânicos) e custo de peças substituídas.
Scripts SQL:
Criação da estrutura lógica do banco de dados (DDL).
Inserção de dados fictícios para testes de sistema.
Consultas SQL para extração de relatórios de desempenho e faturamento.
# Consultas Implementadas:
Listagem de veículos por cliente.
Cálculo de faturamento total por veículo (usando GROUP BY e HAVING).
Ordenação de serviços por valor (usando ORDER BY).
Relatórios detalhados unindo tabelas de clientes e ordens de serviço (JOIN).
